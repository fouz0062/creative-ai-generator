Why This Architecture?

  • Scalable: Serverless architecture handles thousands of concurrent users
  • Cost-Effective: Nearly free to run (~$1-2/month)
  • Secure: Enterprise-grade AWS Cognito authentication
  • Fast: Global CDN via Vercel, optimized Next.js loading
  • Reliable: 99.99% uptime with AWS infrastructure
  • Modern: Latest Next.js 16, TypeScript, Tailwind CSS
  • Developer-Friendly: Easy to maintain, deploy, and scale
  • No Cold Starts: Vercel edge functions are always warm

Infrastructure Management (AWS CDK)

AWS CDK is NOT optional - it's actively used to manage AWS infrastructure as code.

‚úÖ Deployed Stacks (via CDK):
  • ZiaGenDataStack - Creates and manages S3 bucket for image storage
  • ZiaGenCognitoStack - Creates and manages Cognito User Pool for authentication

Deploy command: cdk deploy ZiaGenDataStack ZiaGenCognitoStack

‚ùå Not Deployed:
  • ZiaGenApiStack - Would create:
    • ‚Üí API Gateway (HTTP endpoint receiver)
    • ‚Üí Lambda Functions (API logic executor)
    • ‚Üí Integration between them

üí° Key Point: API Gateway + Lambda are ALWAYS deployed together

In AWS, you can't use Lambda without an HTTP trigger. API Gateway provides that trigger. If you deploy ZiaGenApiStack, you get both services.

Why not deployed? Using Vercel instead:

  • ‚úì Vercel acts as API Gateway (receives HTTP requests)
  • ‚úì Vercel acts as Lambda (runs serverless functions)
  • ‚úì No cold starts, auto-deploy, free tier
  • ‚úì Code stays in app/api/ folder
